# Security Audit Lab

## Boundaries

This project kit is defensive only. It does not provide exploit deployment, evasion, credential theft, persistence, or unauthorized scanning guidance.

## Instructions

1. Confirm scope and authorization first.
2. Choose the least invasive review mode that answers the risk question.
3. Record evidence, impact, likelihood, owner, and remediation.
4. Separate code findings, black-box observations, supply-chain risks, and compliance gaps.
5. Recommend verification tests and monitoring.
