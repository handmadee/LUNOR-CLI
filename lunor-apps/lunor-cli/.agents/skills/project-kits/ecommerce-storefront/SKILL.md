# Ecommerce Storefront

## Instructions

Use this project kit when the task spans multiple ecommerce concerns.

1. Split work into storefront experience, backend API, payment extension, security, and deployment readiness.
2. Keep product, cart, checkout, order, customer, and payment concepts separate.
3. Design checkout as a resilient flow, not a single request.
4. Define SEO, analytics, accessibility, and performance acceptance criteria before UI implementation.
5. Confirm rollback and incident handling for payment or order failures.

## Expected Skill Composition

- `frontend-ui`
- `backend-api`
- `payment-extension`
- `security-appsec`
- `devops-deployment`
