# Design System

## Instructions

1. Separate tokens, primitives, components, patterns, and product-specific compositions.
2. Keep component APIs stable, typed, and accessibility-aware.
3. Document variants, states, constraints, and examples.
4. Avoid one-off visual exceptions unless they become named patterns.
5. Add visual and interaction regression coverage for shared components.
