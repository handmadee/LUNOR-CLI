# AI ML Ops

## Instructions

1. Define task, model contract, eval set, and acceptance threshold.
2. Track prompt/version changes with output examples.
3. Separate offline evals, online monitoring, and human review queues.
4. Add fallback behavior for model errors, latency, rate limits, and safety refusals.
5. Monitor cost, quality, drift, and user-impacting failures.
